# GitHub Portfolio Repo

## Objective

Create a public repository that demonstrates cybersecurity learning without exposing private data.

## Deliverables

- `README.md` with a clear purpose.
- `labs/` folder with one sanitized lab note.
- `scripts/` folder with one harmless helper script or pseudocode.
- `SECURITY.md` describing responsible disclosure boundaries.

## Success Criteria

- The repo is readable by a recruiter or instructor.
- No secrets, internal identifiers, or unsafe exploit details are present.
